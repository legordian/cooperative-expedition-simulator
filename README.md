# Cooperative Expedition Simulator

*A game about maintaining a shared mental model of a complex system under uncertainty.*

## Overview

Many cooperative games do not create genuine cooperation. Players may share a goal, but often solve independent problems in parallel, execute fixed specialist roles, or simply follow the instructions of a single planner.

This project explores a different approach.

The central idea is that cooperation should emerge naturally from complexity, uncertainty, limited attention, and the need to maintain a shared understanding of reality.

Instead of restricting players through classes or information asymmetry, the game aims to create situations where no single player can effectively understand or manage everything at once.

The game is not primarily about operating a vehicle.

The game is about maintaining an accurate model of reality.

## Core Principle

Failure occurs when:

* Reality changes.
* The crew's understanding does not.

Success occurs when:

* The crew continuously updates its shared understanding.
* Plans are adapted to changing conditions.

## Design Goals

* Emergent specialization rather than fixed classes.
* Shared situational awareness.
* Collaborative diagnosis.
* Planning under uncertainty.
* Crew resource management.
* Exploration and expeditions rather than combat.

## Current Status

This repository contains a design exploration rather than an active game project.

The goal is to document and discuss a style of cooperative gameplay that I believe is currently underrepresented.

Feedback, criticism, alternative approaches, and related examples are welcome.
